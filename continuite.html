Continuité sans agents IAContinuity without AI agents

Agentica n'a jamais dépendu d'un agent IA pour fonctionner. Les tokens, les contrats de composants et les règles de gouvernance sont des fichiers versionnés dans Git — jamais enfermés dans une conversation d'agent. Voici le plan documenté et scripté qui le prouve, pour l'équipe qui maintient le système et pour les équipes qui le consomment. Agentica has never depended on an AI agent to function. Tokens, component contracts and governance rules are versioned files in Git — never locked inside an agent conversation. Here is the documented, scripted plan that proves it, for the team that maintains the system and for the teams that consume it.

Le dernier mot reste toujours humainHumans always keep the final word Ce plan ne change aucune règle de gouvernance — tokens-system.md, git-workflow.md, code-style.md restent en vigueur à l'identique. Seul l'exécutant change : un humain fait ce qu'un agent faisait par commodité. This plan changes no governance rule — tokens-system.md, git-workflow.md, code-style.md stay exactly in force. Only the executor changes: a human does what an agent used to do for convenience.

2.3 Garde-fou anti-contournement2.3 Anti-bypass safeguard

L'absence d'agent ne change pas la règleAgent absence doesn't change the rule Sans agent pour faciliter l'usage du design system, une équipe pressée peut être tentée de contourner Agentica entièrement (retour d'expérience documenté chez Spotify/Encore). tokens-system.md reste en vigueur — une valeur en dur reste interdite, elle est juste vérifiée à la main. Si un délai est intenable, la voie correcte est l'escalade (2.4), jamais le contournement silencieux. Without an agent easing design system adoption, a pressured team may be tempted to bypass Agentica entirely (a documented Spotify/Encore case study). tokens-system.md still applies — a hardcoded value is still forbidden, it is just checked by hand. If a deadline is untenable, the correct path is escalation (2.4), never silent bypass.
