# Agentica

## Le système de décisions pour les humains et les agents IA

Les produits numériques accumulent des milliers de décisions : couleurs, espacements, comportements, composants, règles d'accessibilité, conventions techniques, arbitrages architecturaux et processus de gouvernance.

La plupart de ces décisions restent dispersées entre Figma, Storybook, Confluence, GitHub, Slack ou, pire encore, dans la mémoire de quelques experts.

Cette connaissance devient difficile à transmettre, difficile à maintenir et pratiquement impossible à exploiter par des agents IA.

Agentica a été conçu pour résoudre ce problème.

Il transforme les décisions en contrats structurés compréhensibles autant par les humains que par les agents IA, tout en conservant une règle fondamentale :

> Le dernier mot reste toujours humain.

---

# Pourquoi Agentica existe

Les équipes ne souffrent généralement pas d'un manque de composants.

Elles souffrent plutôt :

* d'une perte de connaissances ;
* d'une accumulation de décisions invisibles ;
* d'une dette UX et technique grandissante ;
* d'une documentation qui devient rapidement obsolète ;
* d'une dépendance envers quelques spécialistes ;
* d'une difficulté à intégrer l'intelligence artificielle dans leurs processus sans perdre le contrôle.

Agentica répond à ce problème en créant une source unique de vérité, gouvernée et traçable.

---

# Un système. Deux consommateurs. Zéro compromis.

Les systèmes de design traditionnels sont pensés pour être lus par des humains.

Les systèmes automatisés, eux, nécessitent des règles explicites, structurées et interprétables.

Agentica ne choisit pas entre les deux.

Il permet aux humains et aux agents IA de consommer les mêmes connaissances sans divergence.

### Les humains

* comprennent ;
* décident ;
* approuvent ;
* gouvernent ;
* déploient.

### Les agents IA

* observent ;
* analysent ;
* détectent ;
* proposent ;
* documentent ;
* automatisent.

Les agents ne remplacent pas les humains.

Ils augmentent leur capacité d'action.

---

# Trois niveaux de décisions

Agentica repose sur trois couches complémentaires.

## 1. Les fondations

Les fondations définissent les règles visuelles fondamentales :

* couleurs ;
* typographie ;
* espacements ;
* iconographie ;
* principes d'accessibilité.

Elles constituent le socle du système.

---

## 2. Les contrats sémantiques

Les tokens sémantiques décrivent l'intention plutôt que les valeurs brutes.

Par exemple :

* action primaire ;
* succès ;
* avertissement ;
* danger ;
* surface ;
* texte secondaire.

Cette approche rend les décisions indépendantes des technologies et compréhensibles par les humains comme par les agents IA.

---

## 3. Les composants

Les composants représentent des contrats comportementaux.

Ils définissent :

* leur intention ;
* leurs variantes autorisées ;
* les règles d'accessibilité ;
* les cas limites ;
* les anti-patterns ;
* les tokens qu'ils consomment ;
* leurs contraintes de gouvernance.

Chaque composant est une décision documentée plutôt qu'une simple implémentation visuelle.

# La qualité comme garantie structurelle

La qualité ne devrait pas être une étape finale.

Elle devrait être intégrée au système.

Agentica applique cette philosophie grâce à des pipelines automatisés qui vérifient notamment :

* la cohérence des tokens ;
* la conformité WCAG ;
* les régressions visuelles ;
* la documentation ;
* les décisions architecturales ;
* la qualité des commits ;
* les conventions du système.

Rien n'entre dans le système sans contrôle.

---

# Des décisions traçables

Chaque choix important est documenté sous forme de décision architecturale.

Pour chaque décision, Agentica conserve :

* le contexte ;
* la question posée ;
* la décision retenue ;
* les alternatives rejetées ;
* les conséquences ;
* les coûts acceptés.

Cette mémoire rend le système auditable et transmissible.

Les décisions ne dépendent plus de la mémoire des individus.

---

# Human First. AI Ready.

Agentica a été conçu dès le départ pour fonctionner dans un environnement où humains et agents IA collaborent.

Les agents peuvent :

* détecter des problèmes ;
* générer de la documentation ;
* produire du code ;
* suggérer des corrections ;
* exécuter des validations ;
* accélérer les flux de travail.

Cependant, certaines limites sont non négociables.

Les agents ne peuvent pas :

* approuver leurs propres modifications ;
* contourner les règles de gouvernance ;
* déployer sans validation humaine ;
* inventer des décisions ou des variantes inexistantes.

Le contrôle reste toujours humain.

---

# Une architecture durable

Les technologies évoluent.

Les frameworks changent.

Les outils disparaissent.

Les connaissances, elles, doivent survivre.

Agentica privilégie :

* les standards ouverts ;
* les Web Components ;
* la séparation entre intention et implémentation ;
* les contrats explicites ;
* la portabilité ;
* l'indépendance vis-à-vis des frameworks.

L'objectif n'est pas seulement de construire des interfaces.

L'objectif est de préserver la connaissance et les décisions dans le temps.

---

# Plus qu'un design system

Agentica n'est pas une bibliothèque de composants.

Ce n'est pas non plus une documentation ou un Storybook.

Agentica est un système de décisions.

Il relie le design, l'architecture, la gouvernance, l'accessibilité, l'automatisation et l'intelligence artificielle dans une même source de vérité.

Parce qu'à l'ère des agents IA, la valeur ne réside plus uniquement dans les interfaces.

Elle réside dans les connaissances et les décisions qui permettent de les construire.
