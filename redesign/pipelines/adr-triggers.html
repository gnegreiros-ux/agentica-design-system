Déclencheurs ADRsADR triggers

Bloquer tout commit contenant une décision architecturale nouvelle non documentée par un ADR.Block any commit containing a new architectural decision not documented by an ADR.

Flux du pipelinePipeline flow

01 · Analyse01 · Analysis
Classifier chaque changementClassify each change
Décision nouvelle (= ADR requis) ou application d'une décision existante (= pas d'ADR nécessaire).New decision (= ADR required) or application of an existing decision (= no ADR needed).
02 · Critères02 · Criteria
4 questions de déclenchement4 trigger questions
Décision nouvelle ? · Impact cross-équipe ? · Difficile à défaire ? · Alternatives rejetées ? — 2 "oui" → ADR obligatoire.New decision? · Cross-team impact? · Hard to reverse? · Rejected alternatives? — 2 "yes" → ADR mandatory.
03 · Résultat03 · Outcome
ADR créé ou commit validéADR created or commit cleared
Décision non documentée → commit bloqué jusqu'à création de l'ADR. Décision existante appliquée → commit autorisé.Undocumented decision → commit blocked until ADR created. Existing decision applied → commit cleared.

Valeur métierBusiness value

Aucune décision sans trace. Ce gate distingue les changements qui appliquent les règles existantes de ceux qui en créent de nouvelles — et s'assure que ces derniers sont documentés.No decision without a trace. This gate distinguishes changes that apply existing rules from those that create new ones — and ensures the latter are documented.

ContrôlesChecks

  1. Nouvelle police de caractères → ADR requis (type : Typographie)New typeface → ADR required (type: Typography)

  3. Nouveau composant au système → ADR requis (type : Composant)New system component → ADR required (type: Component)
  4. Nouveau pipeline CI/CD → ADR requis (type : Infrastructure)New CI/CD pipeline → ADR required (type: Infrastructure)
  5. Changement de token sémantique (sens/intention) → ADR requis (type : Token)Semantic token change (meaning/intent) → ADR required (type: Token)
